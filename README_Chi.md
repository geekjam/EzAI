# AI 工作流（.ai）

一套把 AI 协作过程拆分成**计划文件**的工作流。每个阶段对应一个 `.md` 计划文件，在 Claude 里直接点名执行哪个计划，AI 就按该文件的步骤推进。

> 目标：**容易看、可看懂、容易改。**

---

## 工作流总览

把一次开发拆成三个阶段，每个阶段一个计划文件：

| 阶段 | 计划文件 | 做什么 | 产出 |
|---|---|---|---|
| 1. 定义项目 | `.ai/DefineProject.md` | 明确项目目标、技术栈、目录结构、规范 | `README.md`、`FileTree.md`、`Standards.md` |
| 2. 制定任务 | `.ai/MakeWork.md` | 把要做的事拆成步骤、逐项问清楚 | `.ai/task/N.xxTask.md` 任务清单 |
| 3. 执行任务 | `.ai/DoWork.md` | 按任务清单一步步做，做完回写状态 | 代码 + 更新后的任务清单 |

流程：**DefineProject → MakeWork → DoWork →（如有偏差）回到 DefineProject 同步文档**。

---

## 目录结构

```
项目根/
└── .ai/
    ├── README.md          # 本说明文件
    ├── DefineProject.md   # 阶段 1：定义项目
    ├── MakeWork.md        # 阶段 2：制定任务计划
    ├── DoWork.md          # 阶段 3：执行任务计划
    └── task/              # 任务计划文件存放处
        ├── 1.xxTask.md
        └── 2.xxxTask.md
```

## 使用方式

### 第一步：把 `.ai` 文件夹复制到你的项目根目录

直接整个文件夹拷过去就行，不需要改任何配置。

### 第二步：在 Claude 里点名执行某个计划

直接用自然语言告诉 Claude 执行哪个计划文件即可，例如：

- **`执行.ai\DefineProject 计划`**
  开始定义/更新项目，生成 `README.md`、`FileTree.md`、`Standards.md`。

- **`执行.ai\MakeWork 计划`**
  根据当前项目状态制定任务计划，生成 `.ai/task/N.xxTask.md`。
  如果项目代码还没初始化，默认任务就是「初始化项目」。

- **`按.ai\MakeWork 计划新增功能 xxx`**
  带着具体需求制定任务计划，把「新增功能 xxx」拆成可执行步骤。

- **`执行.ai\DoWork 计划`**
  按 `.ai/task/` 下的任务清单逐项执行，做完更新状态。
  全部完成后会自动回头执行 `DefineProject`，把架构/框架版本等变更同步回文档。

> 计划文件本身就是给 AI 看的指令。Claude 读到「执行 X 计划」时，会打开 `.ai/X.md` 并按里面写的步骤走。

## 任务清单状态约定

任务计划 `.md` 文件里每一项前用状态标记：

| 标记 | 含义 |
|---|---|
| `[ ]` | 未开始 |
| `[~]` | 进行中 |
| `[x]` | 已完成 |

文件命名：`数字 + 任务名`，例如 `1.初始化项目`、`2.用户登录模块`。

---

## 怎么改

这套工作流没有任何代码，全是 Markdown。改计划行为就直接编辑 `.ai/` 下的三个 `.md` 文件：

- 想改**定义项目**问什么 → 编辑 `DefineProject.md`
- 想改**制定任务**的方式 → 编辑 `MakeWork.md`
- 想改**执行任务**的流程 → 编辑 `DoWork.md`

任务计划模板在 `MakeWork.md` 里描述，改那里的命名/状态约定即可影响所有新生成的任务文件。

---

## 设计要点

1. **每个阶段一个文件** —— 职责单一，看一眼就知道这步干什么。
2. **计划文件即指令** —— 不需要脚本、不需要配置，Claude 直接读 Markdown 执行。
3. **强制提问对齐** —— `DefineProject` 和 `MakeWork` 都要求 AI 一问一答问到清楚再动手，避免理解偏差。
4. **执行完回写文档** —— `DoWork` 收尾时会跑一次 `DefineProject`，确保文档和代码不脱节。
5. **不碰代码直到该碰** —— 前两个阶段明确禁止改代码，先把项目和任务想清楚。
